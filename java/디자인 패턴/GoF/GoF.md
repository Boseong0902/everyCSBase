## 생성 패턴

객체를 직접 new키워드로 생성하면 클래스 간의 결합도가 극도로 높아진다.

따라서 객체 생성 과정을 추상화하여, 구체 클래스에 의존하지 않는것을 목적으로 하는 패턴

- [[싱글톤]] - 해당 클래스의 객체는 전역에서 단 하나만 생성해놓고, 전역에서 접근이 가능하도록 설정한다
- [[정적 팩토리 메서드와 팩토리 메서드]] - 객체를 사용하는 쪽에서 직접 생성하는 것이 아닌, 생성 책임을 다른 곳으로 분리하는 패턴(IoC / DI)
- [[빌더]]

## 구조 패턴

클래스와 객체를 조합하여 더 큰 구조를 만드는 방법에 대한 패턴

- [[어댑터]] - 외부 클래스등을 사용할 때, 자체 비지니스 로직에서 요하는 메서드 시그니처와 다른 경우, 그 사이에 어댑터 클래스를 껴넣음으로써, 확장성을 확보할 수 있는 패턴
- [[프록시]] - 특정 클래스의 객체들에 접근하기 전의 공통 동작들을 클래스 본체와 구분하는 패턴(AOP)
- [[데코레이터]] - 기존 객체를 동일한 인터페이스를 구현한 다른 객체로 감싸서, 기능을 동적으로 추가하는 패턴

## 행위 패턴

객체나 클래스 사이의 상호작용과 책임 분배 방법을 정의한 패턴

- [[전략]] - 상황에 따라 달라질 수 있는 알고리즘이나 정책들을 객체화 하여 동적으로 갈아끼울 수 있는 패턴
- [[템플릿 메서드]] - 전체적인 순서는 부모 클래스에서, 그 세부 작업들은 자식 클래스에서 정의하는 패턴(JDBCTemplate, RestTemplate)
- [[옵저버]] - 어떤 객체의 상태가 바뀌었을 때, 그 객체를 구독하고 있는 객체들에게 자동으로 알림을 보내는 패턴(EventListener & TransactionalEventListener)
