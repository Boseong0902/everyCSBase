# 빌더

복잡한 객체 생성을 단계별로 나누고, 읽기 좋은 방식으로 객체를 만들 수 있게 하는 생성 패턴

생성자 파라미터가 너무 많거나 선택값(필수가 아닌 값)이 많은 객체를 만들 때 사용

그래서 일부 필드만 파라미터로 받아서 초기화시키는 생성자를 만들기도 하지만,

그러면 객체가 불완전한 상태로 존재할 수도 있고, setter를 열어두기 때문에 이후에도 값이 계속 바뀔 수 있다 -> 불변 객체 생성이 어려움

## 빌더 패턴

```java
Member member = new Member.Builder()
    .name("Kim")
    .age(20)
    .email("test@gmail.com")
    .address("Seoul")
    .marketingAgreed(true)
    .build();
```

어떻게 이게 되는지 내부적으로 뜯어보면,

Builder메서드는 값을 세팅한 후, this키워드로 해당 객체를 반환함

따라서 다시 그 객체에 대해 Builder메서드를 호출할 수 있음
