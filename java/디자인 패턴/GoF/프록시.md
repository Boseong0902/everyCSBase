# 프록시

프록시 패턴은 특정 객체에 직접 접근하지 않고 대리 객체를 거치게 해서, 실제 객체 호출 전후에 권한 검사, 지연 로딩, 캐싱, 로깅, 트랜잭션 같은 제어 로직을 추가하거나 접근 조건을 통제하기 위한 패턴이다

## 언제?

- 지연 로딩 -> 실제 객체 생성 시점을 늦추고 싶다
- 권한 검사 -> 실제 객체에 접근해도 되는지 검사하고 싶다
- 캐싱 -> 매번 실제 객체를 호출하고 싶지 않다
- 원격 프록시 -> 실제 객체가 원격에 있는데 로컬 객체처럼 쓰고싶다
- 공통 처리 -> 실제 메서드 호출 전후에 공통 처리를 넣고싶다
	- 트랜잭션, 로깅, 성능 측정 등등

## vs 데코레이터

프록시 -> 실제 객체에 대한 접근을 제어한다

[[데코레이터]] -> 기존 객체를 감싸서 기능을 동적으로 추가한다
