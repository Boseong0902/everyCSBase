# 데코레이터

기존 객체를 동일한 인터페이스를 구현한 다른 객체로 감싸서, 기능을 동적으로 추가하는 구조 패턴

즉, 기존 객체에 기능을 추가하는, 확장과 조합이 목적이다

## vs 프록시

[[프록시]] -> 실제 객체에 대한 접근을 제어한다

데코레이터 -> 같은 인터페이스를 유지한 채 기능을 확장/조합한다

## 예시

알림 기능을 하는 객체가 필요하다고 해보자

```java
public interface Notifier {
    void send(String message);
}

public class EmailNotifier implements Notifier {
    @Override
    public void send(String message) {
        System.out.println("이메일 전송: " + message);
    }
}
```

여기서 sms, slack, telegram, kakaotalk 으로 보내는 기능이 추가되고, 이들을 상황에 따라 섞어야된다면? -> 경우의 수는 31가지가 된다

- 31가지의 구체 클래스로 상속할 순 없으니, 상황에 따라 껴넣을 수 있는 필터같은게 필요하다

먼저 공통 데코레이터를 만든다

```java
public abstract class NotifierDecorator implements Notifier {
    protected final Notifier notifier;

    protected NotifierDecorator(Notifier notifier) {
        this.notifier = notifier;
    }

    @Override
    public void send(String message) {
        notifier.send(message);
    }
}

public class SmsNotifierDecorator extends NotifierDecorator {
    public SmsNotifierDecorator(Notifier notifier) {
        super(notifier);
    }

    @Override
    public void send(String message) {
        super.send(message);
        sendSms(message);
    }

    private void sendSms(String message) {
        System.out.println("SMS 전송: " + message);
    }
}

// 이런 식으로 구현한 다음,

Notifier notifier = new SlackNotifierDecorator(
                        new SmsNotifierDecorator(
                            new EmailNotifier()
                        )
                    );
// 이런 식으로 매개변수로 주입해주면 된다
```

그리고 만약 이런 데코레이터 조합이 동적으로 바뀌어야 한다면, 팩토리 패턴과 같이 사용할 수도 있다
