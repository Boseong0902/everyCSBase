# JVM

JVM은 Java 바이트코드를 실행하는 가상 머신이다.

Java 소스코드는 컴파일러를 통해 `.class` 바이트코드로 변환되고, JVM은 이 바이트코드를 각 OS 위에서 실행한다. 이 구조 때문에 Java 코드는 OS별로 다시 컴파일하지 않아도 JVM이 있는 환경에서 실행될 수 있다.

## 실행 흐름

1. 자바 컴파일러가 `.java` 파일을 `.class` 파일로 변환한다.
2. 클래스 로더가 `.class` 파일을 읽어 JVM 메모리 영역에 로딩한다.
3. 실행 엔진이 바이트코드를 해석하거나 JIT 컴파일해서 실행한다.
4. 실행 중 JVM은 스레드 동기화와 [[Garbage Collection]] 같은 런타임 관리를 수행한다.

## Runtime Data Area

- Method Area: 클래스 구조, 메서드 바이트코드, `static` 변수, 런타임 상수 풀을 저장한다. 모든 스레드가 공유한다.
- Heap: `new`로 생성한 객체와 배열을 저장한다. 모든 스레드가 공유한다.
- Stack: 지역 변수, 매개변수, 메서드 호출 정보를 저장한다. 스레드별로 독립적이다.
- PC Register: 현재 실행 중인 명령어 주소를 저장한다. 스레드별로 독립적이다.
- Native Method Stack: 네이티브 메서드 호출 정보를 저장한다. 스레드별로 독립적이다.
