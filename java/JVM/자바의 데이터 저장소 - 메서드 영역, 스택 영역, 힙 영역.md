# 자바의 데이터 저장소 - 메서드 영역, 스택 영역, 힙 영역

자바 프로그램이 실행되면 JVM은 실행에 필요한 데이터를 여러 런타임 메모리 영역에 나누어 저장한다.

대표적으로 메서드 영역, 스택 영역, 힙 영역을 구분해서 이해하면 된다.

## 메서드 영역

클래스 수준의 정보가 저장되는 영역이다.

주로 다음 정보가 저장된다.

- 클래스 이름
- 부모 클래스 정보
- 메서드 정보
- 필드 정보
- static 변수
- 런타임 상수 풀

클래스가 로딩될 때 JVM이 클래스 메타데이터를 메서드 영역에 올린다.

객체마다 따로 존재하는 정보가 아니라, 클래스 단위로 공유되는 정보가 들어간다.

따라서 `static` 변수나 `static` 메서드는 특정 객체에 속한 것이 아니라 클래스에 속한 것으로 이해하면 된다.

## 스택 영역

메서드 호출마다 생성되는 스택 프레임이 저장되는 영역이다.

메서드가 호출되면 해당 메서드의 스택 프레임이 쌓이고, 메서드 실행이 끝나면 제거된다.

스택 프레임에는 주로 다음 정보가 들어간다.

- 지역 변수
- 매개변수
- 임시 연산 값
- 메서드 실행 흐름 정보

기본형 지역 변수는 실제 값이 스택 영역에 저장된다.

참조형 지역 변수는 객체 자체가 아니라, 힙 영역에 있는 객체를 가리키는 참조값이 스택 영역에 저장된다.

## 힙 영역

객체와 배열이 저장되는 영역이다.

`new` 키워드로 생성한 객체는 힙 영역에 저장된다.

힙 영역에 있는 객체는 여러 스택 프레임의 참조 변수가 함께 가리킬 수 있다.

더 이상 어떤 참조 변수도 접근할 수 없는 객체는 가비지 컬렉터의 수집 대상이 된다.

## 예시

```java
public class Person {
    static int count = 0;

    private final String name;

    public Person(String name) {
        this.name = name;
        count++;
    }

    public void introduce() {
        String message = "hello " + name;
        System.out.println(message);
    }
}
```

위 코드에서 메모리 관점은 다음과 같다.

- `Person` 클래스 정보는 메서드 영역에 저장된다
- `count`는 클래스에 속한 `static` 변수이므로 메서드 영역 쪽에서 관리된다
- `new Person("kim")`으로 생성한 객체는 힙 영역에 저장된다
- `name` 필드는 힙 영역의 `Person` 객체 내부에 존재한다
- `introduce()`가 호출되면 스택 영역에 스택 프레임이 생성된다
- `message` 지역 변수는 `introduce()`의 스택 프레임 안에서 관리된다

## 정리

| 영역 | 저장 대상 | 생명주기 |
| --- | --- | --- |
| 메서드 영역 | 클래스 정보, static 변수, 런타임 상수 | 클래스 로딩부터 언로딩까지 |
| 스택 영역 | 메서드 호출 정보, 지역 변수, 매개변수 | 메서드 호출부터 종료까지 |
| 힙 영역 | 객체, 배열, 인스턴스 필드 | 객체 생성부터 GC 수집까지 |
