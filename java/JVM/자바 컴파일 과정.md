# 자바 컴파일 과정

자바는 JVM 덕분에 OS에 독립적으로 실행된다.

소스코드가 곧바로 OS별 기계어로 변환되는 것이 아니라, JVM이 이해하는 바이트코드라는 중간 단계를 거친다. 그래서 각 OS에 맞는 JVM만 있으면 같은 `.class` 파일을 실행할 수 있다.

## 컴파일부터 실행까지

1. Java 소스코드를 작성한다.
2. 자바 컴파일러가 소스코드를 `.class` 파일로 변환한다.
3. 컴파일된 `.class` 파일을 JVM의 [[클래스 로더]]가 읽는다.
4. 클래스 로더는 필요한 클래스를 동적 로딩해서 [[자바의 데이터 저장소 - 메서드 영역, 스택 영역, 힙 영역|Runtime Data Area]]에 적재한다.
5. 실행 엔진이 메모리에 올라온 바이트코드를 명령어 단위로 실행한다.

## 실행 엔진의 방식

### 인터프리터

바이트코드를 한 줄씩 읽어서 해석하고 실행한다.

- 별도 네이티브 컴파일 과정이 없어 프로그램 시작이 빠르다.
- 같은 코드가 반복 실행되면 매번 해석해야 해서 비효율적이다.

### JIT 컴파일러

반복 실행이 많은 바이트코드를 네이티브 코드로 컴파일하고, 이후에는 컴파일된 코드를 직접 실행한다.

- 인터프리터의 반복 해석 비용을 줄이기 위한 방식이다.
- 일반 코드는 인터프리터로 실행하다가, 반복 횟수가 임계치를 넘는 코드는 별도 스레드에서 비동기적으로 컴파일한다.
