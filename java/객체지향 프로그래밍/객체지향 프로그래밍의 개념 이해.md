## 객체 지향 프로그래밍 관련 개념 답변 방법

결국 객체 지향 프로그래밍은 실세계를 코드상으로 옮기기 위한 노력의 결과

따라서 이를 설명하기 위한 최선의 방법은 현실 세계에 빗대어 설명하는 것

## 객체란?

### 키워드

- 객체는 실세계에서의 개체이다
- 객체는 상태(필드)와 동작(메서드)을 갖는다
- 객체는 클래스의 인스턴스를 나타낸다
- 객체는 메모리에서 공간을 차지한다
- 객체는 다른 객체와 소통할 수 있다

### 추가 사실

- 접근 제어자를 통해 가시성 범위를 제한할 수 있음
- 가변 속성 또는 불변 속성을 갖는다
- 가비지 컬렉터에 의해 수집된다
- 객체는 [[자바의 데이터 저장소 - 메서드 영역, 스택 영역, 힙 영역]] 중 힙 영역에 저장된다

## 클래스란?

### 키워드

- 클래스는 객체를 생성하기 위한 템플릿
- 클래스는 인스턴스화하기 전까진 메모리의 힙 영역을 소모하지 않는다
- 클래스는 여러번 인스턴스화 할 수 있음
- 하나의 클래스는 하나의 작업만 한다([[SRP - 단일 책임]])
- 클래스 수준의 정보와 [[static, final 키워드|static]] 멤버는 메서드 영역 쪽에서 관리된다

## 추상화란?

### 키워드

사용자가 작업하기 위한 부분만 노출시키고, 나머지는 숨긴다

이를 통해 사용자는 애플리케이션이 일을 수행하는 원리가 아니라, 애플리케이션이 수행하는 일 자체에 집중할 수 있다

- 복잡성을 낮추고
- 코드의 재사용성을 높이며
- 코드 중복을 방지하고
- 낮은 결합도와 높은 응집도를 유지한다

## 캡슐화?

객체 상태(필드)를 외부로부터 숨긴 상황에서 이 상태에 접근하는 일련의 공개 메서드만 노출하는 기법

각 객체가 클래스 내에서 객체의 상태를 비공개로 유지할 때 성립

이를 통해 느슨한 결합, 코드의 재사용성, 정보 은닉 등을 달성할 수 있다

## 상속이란?

코드의 재사용성

## 다형성이란?

상황에 따라 객체가 다르게 동작할 수 있도록 하는 특징

부모 클래스 혹은 인터페이스에서 선언한 메서드를 오버라이딩을 하거나,

혹은 한 메서드에 대해 오버로딩하는 방법으로 구현할 수 있다

### 오버로딩 vs 오버라이딩

- 오버로딩 - 입력받는 매개변수의 종류에 따라 동작이 달라짐
- 오버라이딩 - 주입받는 객체의 종류에 따라 동작이 달라짐
- 오버로딩의 경우, 매개변수의 타입이 이미 컴파일 시점에 정해져있음. -> 즉, 정적 디스패치가 이루어짐
- 오버라이딩의 경우, 주입되는 객체는 동적으로 주입되므로, 컴파일 시점엔 어떤 동작을 할 지 모름 -> 동적 디스패치

자세한 차이는 [[정적 다형성과 런타임 다형성]]에서 정리한다.
