# 정적 다형성과 런타임 다형성

다형성은 상황에 따라 같은 이름의 기능이 다르게 동작할 수 있도록 하는 특징이다.

자바에서는 크게 정적 다형성과 런타임 다형성으로 나눠볼 수 있다.

## 정적 다형성

정적 다형성은 컴파일 시점에 어떤 메서드가 호출될지 결정되는 다형성이다.

대표적인 예시는 오버로딩이다.

상세 규칙은 [[오버라이딩 vs 오버로딩]]에서 따로 정리한다.

```java
public class Printer {
    public void print(String value) {
        System.out.println(value);
    }

    public void print(int value) {
        System.out.println(value);
    }
}
```

```java
Printer printer = new Printer();
printer.print("hello");
printer.print(10);
```

위 코드에서 `print("hello")`는 `String` 매개변수를 받는 메서드로, `print(10)`은 `int` 매개변수를 받는 메서드로 컴파일 시점에 결정된다.

즉, 입력받는 매개변수의 타입과 개수에 따라 호출 대상이 정해진다.

## 런타임 다형성

런타임 다형성은 실행 시점에 실제 객체 타입에 따라 어떤 메서드가 호출될지 결정되는 다형성이다.

대표적인 예시는 오버라이딩이다.

상세 규칙은 [[오버라이딩 vs 오버로딩]]에서 따로 정리한다.

```java
public interface PaymentModule {
    void pay();
}

public class CardPaymentModule implements PaymentModule {
    @Override
    public void pay() {
        System.out.println("카드 결제");
    }
}

public class FakePaymentModule implements PaymentModule {
    @Override
    public void pay() {
        System.out.println("가짜 결제");
    }
}
```

```java
public class PaymentService {
    private final PaymentModule paymentModule;

    public PaymentService(PaymentModule paymentModule) {
        this.paymentModule = paymentModule;
    }

    public void pay() {
        paymentModule.pay();
    }
}
```

`PaymentService`는 `PaymentModule` 타입만 알고 있다.

하지만 실제 실행 시점에 주입된 객체가 `CardPaymentModule`인지 `FakePaymentModule`인지에 따라 호출되는 `pay()` 구현이 달라진다.

## 오버로딩과 오버라이딩

| 구분 | 오버로딩 | 오버라이딩 |
| --- | --- | --- |
| 다형성 종류 | 정적 다형성 | 런타임 다형성 |
| 결정 시점 | 컴파일 시점 | 실행 시점 |
| 기준 | 메서드 이름 + 매개변수 타입/개수 | 실제 객체 타입 |
| 대표 관계 | 같은 클래스 안의 같은 이름 메서드 | 부모 타입과 자식 타입 |
| 디스패치 | 정적 디스패치 | 동적 디스패치 |

## 정적 디스패치

정적 디스패치는 컴파일 시점에 호출할 메서드가 정해지는 방식이다.

오버로딩은 매개변수 타입을 보고 컴파일러가 호출 대상을 결정한다.

```java
public void print(Object value) {
    System.out.println("object");
}

public void print(String value) {
    System.out.println("string");
}
```

```java
String value = "hello";
print(value); // String 버전 호출
```

컴파일러는 `value`의 선언 타입이 `String`인 것을 보고 `print(String value)`를 선택한다.

## 동적 디스패치

동적 디스패치는 런타임 시점에 실제 객체를 보고 호출할 메서드가 정해지는 방식이다.

```java
PaymentModule paymentModule = new CardPaymentModule();
paymentModule.pay();
```

변수의 타입은 `PaymentModule`이지만, 실제 객체는 `CardPaymentModule`이다.

그래서 실행 시점에는 `CardPaymentModule`의 `pay()`가 호출된다.

## 왜 중요한가?

정적 다형성은 같은 기능 이름을 매개변수 형태에 따라 다르게 제공할 수 있게 해준다.

런타임 다형성은 호출부가 구체 클래스를 몰라도 기능을 사용할 수 있게 해준다.

이 때문에 런타임 다형성은 [[OCP - 개방-폐쇄]]와 직접 연결된다.

새 구현체가 추가되어도 호출부는 부모 타입이나 [[interface]]에만 의존하면 되기 때문에 기존 코드를 덜 수정하게 된다.

## 주의할 점

오버로딩은 다형성이지만, 객체가 바뀌어서 동작이 바뀌는 구조는 아니다.

객체지향 설계에서 말하는 다형성의 핵심은 보통 런타임 다형성에 더 가깝다.

즉, 부모 타입이나 인터페이스 타입으로 사용하고, 실제 구현체의 오버라이딩된 메서드가 실행되는 구조가 핵심이다.
