# OOP 4대 원칙

OOP 4대 원칙은 추상화, 다형성, 캡슐화, 상속이다.

## 추상화

중요한 특징만 드러내고 불필요한 세부 구현은 숨김으로써, 복잡도를 낮추고 역할만을 보여줌으로써 재사용을 쉽게 만듬

같은 요청을 했을 때, 주입된 객체에 따라 다르게 동작

제공자/설계자의 입장에서 어떤 계약을 만들 것인가를 정하는 과정

인터페이스와 추상클래스로 구현

## 다형성

같은 메서드가 객체에 따라 다르게 동작

무엇을 할 수 있는가를 정의해놓는 것

사용자/호출자 쪽에서 그 계약만 믿고 유연하게 사용할 수있게 설계하는 과정

오버라이딩과 오버로딩 기법으로 구현

## 캡슐화

객체의 상태를 외부에서 직접 접근하지 못하게 숨기고, 공개된 메서드를 통해서만 상태를 다루게 하는 원칙

필드는 `private`으로 감추고, 필요한 동작만 메서드로 공개하는 식으로 구현한다.

이를 통해 객체 내부 구현이 바뀌더라도 외부 호출부가 받는 영향을 줄일 수 있다.

캡슐화의 핵심은 단순히 필드를 숨기는 것이 아니라, 객체가 자기 상태를 스스로 관리하도록 만드는 것이다.

## 상속

상위 클래스의 필드와 메서드를 하위 클래스가 물려받아 재사용하거나 확장하는 원칙

공통 동작을 부모 클래스에 두고, 구체적인 동작은 자식 클래스에서 오버라이딩할 수 있다.

다만 상속은 부모와 자식 사이의 결합을 강하게 만들 수 있다.

따라서 공통 구현을 재사용하려는 목적만으로 상속을 남용하기보다는, 자식이 부모를 대체할 수 있는 관계인지 함께 봐야 한다.
