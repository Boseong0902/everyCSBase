# 상속보다 컴포지션

컴포지션은 상속 대신 기존 클래스의 인스턴스를 `private` 필드로 품어서 기능을 재사용하는 방식이다.

## 상속의 단점

- 자식 클래스가 부모 클래스의 내부 구현에 의존하게 된다.
- 부모 구현이 바뀌면 자식 클래스가 깨질 수 있다.
- Java는 클래스 다중 상속을 지원하지 않는다.

상속은 부모의 코드와 동작 로직까지 받아오는 구조라 결합이 강해질 수 있다.

## 구현과 인터페이스

인터페이스 구현은 메서드 시그니처라는 약속만 받아온다.

실제 로직은 구현 클래스가 직접 작성한다. 그래서 상속보다 약한 결합을 만들기 쉽다.

## 추상 클래스와 인터페이스

추상 클래스는 공통 상태를 가질 수 있다.

반면 인터페이스는 인스턴스 필드를 가질 수 없다. 인터페이스의 필드는 `public static final` 상수로만 선언된다.

즉, 인터페이스는 상태 공유가 아니라 여러 클래스가 같은 메서드 시그니처를 공유하기 위한 장치에 가깝다.
