# 인터페이스와 추상 클래스

## default 메서드

기존엔 interface내에서 추상 메서드 외의 메서드 정의는 불가능했으나, 자바 8부터 가능해짐 - default method

기존 인터페이스와 구현 체계를 무너뜨리지 않고, 기능을 추가하기 위한 목적

## default vs static

오버라이드 가능 여부로 갈림(static 은 오버라이딩 불가능)

호출 방식의 차이도 있음 -> static 메서드는 객체가 아닌, 클래스/인터페이스에 종속되기 때문에 인터페이스명.메서드명 으로 호출

```java
interface MyInterface {
    static void printStaticHello() {
        System.out.println("Hello from Interface Static!");
    }

    default void printDefaultHello() {
        System.out.println("Hello from Interface Static!");
    }
}

class MyClass implements MyInterface {
    // 구현 클래스 바디
}

public class Main {
    public static void main(String[] args) {
        // 올바른 호출 방법 (인터페이스 명으로 직접 호출)
        MyInterface.printStaticHello();

        // 컴파일 에러 발생 (구체 클래스 명으로 호출 불가)
        // MyClass.printStaticHello();

        // 컴파일 에러 발생 (구체 클래스 인스턴스로 호출 불가)
        MyClass obj = new MyClass();
        // obj.printStaticHello();

        // 올바른 호출 방법 (객체명으로 직접 호출)
        obj.printDefaultHello();

        // 올바른 호출 방법 (참조변수로 호출)
        MyClass myclass = new MyClass();
        myclass.printDefaultHello();

        // p.s 사실 이 예시처럼 new를 통한 클래스 내에서의 객체 생성은 안티패턴
        // 매개변수로 객체를 주입받는것이 좋은 패턴이다
    }
}
```

## 인터페이스와 추상 클래스의 차이

그럼 이렇게 상속 가능하고, 구현도 해놓은 메서드도 있는데, 과연 인터페이스랑 추상 클래스의 차이점은 뭘까?

## 생성자 사용 여부

생성자 -> 상태를 기본 세팅값으로 세팅해주는 함수

추상 클래스는 상태, 즉 필드를 가질 수 있지만, 인터페이스는 static제외 불가능

따라서 인터페이스는 고유의 상태가 없으므로 생성자도 없음 -> 즉, 인터페이스는 스스로 어떤 것도 하지 않는 완전한 추상화를 목표로 함

## 인터페이스 안티패턴

### default메서드를 다른 목적으로 사용

인터페이스에 비지니스 로직을 구현하고, 이를 다중 구현을 통해 클래스에 욱여넣음

-> 즉, 비지니스 로직 구현 및 다중 구현(상속) 남용

구체클래스에서 구현해야될 메서드를 인터페이스 default메서드로 구현

### default메서드만 사용

## 총 정리

위에서도 언급한 내용이라 겹치는 부분이 있지만, 총 정리를 위해 한번 더 언급

### 생성자 사용 여부

생성자 -> 상태를 기본 세팅값으로 세팅해주는 함수

추상 클래스는 상태, 즉 필드를 가질 수 있지만, 인터페이스는 static제외 불가능

따라서 인터페이스는 고유의 상태가 없으므로 생성자도 없음 -> 즉, 인터페이스는 스스로 어떤 것도 하지 않는 완전한 추상화를 목표로 함

### 다중 상속 가능 여부

인터페이스는 여러 개를 구현할 수 있지만, 추상 클래스는 클래스 상속 구조 안에서 하나만 상속할 수 있다.

인터페이스와 추상 클래스 모두 final은 불가능

final -> 서브클래스화 되거나 상속될 수 없게 만드는 키워드
