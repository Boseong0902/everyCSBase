# Java 동시성

Java 동시성은 여러 스레드가 같은 프로세스 안에서 동시에 실행될 때 생기는 문제를 다루는 영역이다.

이 문서는 동시성 주제의 인덱스다. 세부 설명은 각 주제 문서로 나눈다.

## 학습 흐름

1. 여러 작업을 동시에 실행하기 위해 [[스레드 활용 - 멀티 스레딩|스레드]]를 사용한다.
2. 여러 스레드가 같은 데이터를 공유할 때 생기는 [[동시성 문제 - 원자성, 가시성, Race Condition|동시성 문제]]를 이해한다.
3. 공유 데이터 정합성을 지키기 위해 [[synchronized, volatile 키워드|synchronized와 volatile]]을 구분한다.
4. `synchronized`가 실제로 사용하는 [[고유락]]과 재진입, `wait()`/`notify()`를 이해한다.
5. 공유 상태를 줄이기 위해 [[불변 객체]], 지역 변수, 스레드 안전 컬렉션 같은 대안을 고려한다.

## 주제별 문서

- [[스레드 활용 - 멀티 스레딩]]: 스레드 생성 방식, `run()`과 `start()`, 스레드 상태
- [[동시성 문제 - 원자성, 가시성, Race Condition]]: race condition, 원자성, 메모리 가시성
- [[synchronized, volatile 키워드]]: Java 키워드가 보장하는 것과 보장하지 않는 것
- [[고유락]]: 객체 모니터 락, 재진입, 구조적 락, 명시적 락, `wait()`/`notify()`

## 언제 무엇을 쓰는가

| 상황 | 선택 |
| --- | --- |
| 단순 임계 영역 보호 | `synchronized` |
| 단순 상태 플래그의 변경 전파 | `volatile` |
| 증가, 감소 같은 복합 연산 | `synchronized` 또는 원자 클래스 |
| 조건별 대기 큐가 필요함 | `ReentrantLock`과 `Condition` |
| 여러 스레드에서 공유할 문자열 변경 | `StringBuffer` |
| 여러 스레드에서 Map 공유 | `ConcurrentHashMap` |
| 가능한 경우 상태 변경 자체를 줄이고 싶음 | [[불변 객체]] |

## 주의할 점

동시성 문제는 도구부터 외우면 흐름이 끊긴다.

먼저 공유 상태가 있는지 확인하고, 그 공유 상태에서 원자성 또는 가시성 문제가 생기는지 판단한 뒤, 필요한 만큼만 락이나 동시성 도구를 적용하는 순서로 보는 것이 좋다.
