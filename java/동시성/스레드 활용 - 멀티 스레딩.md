# 스레드 활용 - 멀티 스레딩

Java에서 스레드를 만드는 대표적인 방법은 두 가지다.

- `Runnable` 인터페이스 구현
- `Thread` 클래스 상속

대부분은 `Runnable` 구현 방식을 많이 사용한다. Java는 클래스 단일 상속만 가능하기 때문에 `Thread`를 상속하면 다른 클래스를 상속하기 어렵다.

## run()과 start()

`Runnable`을 구현하든 `Thread`를 상속하든 실제 작업 내용은 `run()`에 작성한다.

하지만 스레드를 새로 시작하려면 `start()`를 호출해야 한다.

- `run()`: 현재 스레드의 콜 스택에 메서드를 추가해서 실행한다.
- `start()`: 새로운 스레드를 만들고, 그 스레드의 콜 스택에서 `run()`을 실행한다.

## 스레드 상태

- NEW: OS 수준에서 실행 가능한 스레드가 아직 생성되기 전이다.
- RUNNABLE: 실행 중이거나 실행 가능한 상태다.
- BLOCKED: 동기화 락 때문에 일시 정지된 상태다.
- WAITING: 다른 스레드의 알림이나 조건을 기다리는 상태다.
- TERMINATED: 작업이 종료된 상태다.

## 동기화

`synchronized`는 동시성 처리를 위한 뮤텍스 락이다.

`wait()`와 `notify()`는 `synchronized` 객체에서 조건 변수처럼 사용할 수 있다. 락은 잡았지만 조건이 맞지 않아 효율적인 실행이 불가능한 경우, `wait()`로 대기하고 다른 스레드가 `notify()`로 깨울 수 있다.

생산자-소비자 구조에서 상품이 0개인데 소비자가 락을 잡으면 busy waiting이나 비효율이 생길 수 있다. 이때 소비자가 `wait()`하고 생산자가 상품을 만든 뒤 `notify()`하면 더 효율적으로 진행할 수 있다.
