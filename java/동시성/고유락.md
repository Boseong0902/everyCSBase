# 고유락

Java의 락은 각 객체별로 부여된다.

`synchronized` 인스턴스 메서드는 해당 객체의 고유 락을 사용한다.

## 재진입

Java의 고유 락은 재진입 가능하다.

동일 객체에 대한 락을 이미 가진 스레드는 같은 객체의 다른 `synchronized` 메서드에 다시 진입할 때 락을 새로 얻는 과정이 필요하지 않다.

```java
class Reentrancy {
    public synchronized void a() {
        b();
    }

    public synchronized void b() {
    }
}
```

락은 호출 단위가 아니라 스레드 단위로 소유된다.

## 구조적 락

`synchronized` 키워드를 사용하는 방식이다.

- 객체의 고유 락을 이용한다.
- 락 관리를 JVM이 자동으로 처리한다.
- 락 해제는 락 획득 구조를 벗어날 때 자동으로 이루어진다.

## 명시적 락

`ReentrantLock`처럼 Java 패키지가 제공하는 별도 락 객체를 사용하는 방식이다.

- 개발자가 `lock()`, `unlock()` 시점을 지정한다.
- 조건별 대기 큐를 분리할 수 있다.

구조적 락은 `wait()`와 `notify()`를 위한 대기 큐가 하나다. 명시적 락은 조건별로 대기 큐를 분리할 수 있어 생산자/소비자 같은 상황에서 더 세밀하게 제어할 수 있다.
