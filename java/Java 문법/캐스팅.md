# 캐스팅

캐스팅은 타입 변환이다.

객체지향 문맥에서는 부모 타입과 자식 타입 사이의 업캐스팅, 다운캐스팅을 구분해서 이해해야 한다.

## 업캐스팅

자식 객체를 부모 참조 변수에 넣어서 사용하는 것이다.

```java
class Animal {}
class Dog extends Animal {}

Animal a = new Dog();
```

자식 객체는 부모의 구조를 상속받았기 때문에 부모 타입으로 바라보는 것은 자연스럽다.

## 다운캐스팅

부모 타입 참조 변수를 자식 타입 참조 변수로 다시 변환하는 것이다.

```java
Animal a = new Dog();
Dog d = (Dog) a;
```

이 경우 `a`가 실제로 가리키는 객체가 `Dog`이기 때문에 다운캐스팅이 가능하다. 만약 실제 객체가 `Animal`이나 다른 하위 타입이면 실패한다.

다운캐스팅을 하면 자식 클래스의 멤버에 접근할 수 있다.

## 다운캐스팅을 쓰는 상황

여러 하위 타입을 부모 타입 컬렉션에 담아두고, 특정 타입일 때만 필요한 기능을 사용해야 할 때 사용할 수 있다.

```java
for (Animal a : animals) {
    if (a instanceof Dog) {
        Dog d = (Dog) a;
        d.fetch();
    }
}
```

외부 코드나 레거시 코드처럼 내가 타입 구조를 다형적으로 바꾸기 어려운 경우에도 사용할 수 있다.

## instanceof와 다운캐스팅의 한계

`instanceof` 분기가 많아지면 새 타입이 추가될 때 기존 코드를 계속 수정해야 한다.

```java
void doAction(Animal a) {
    if (a instanceof Dog) {
        ((Dog) a).bark();
    } else if (a instanceof Cat) {
        ((Cat) a).purr();
    }
}
```

이런 구조는 확장성이 떨어진다. 가능한 경우에는 [[정적 다형성과 런타임 다형성|다형성]]과 [[오버라이딩 vs 오버로딩|오버라이딩]]으로 해결하는 편이 낫다.

```java
abstract class Animal{
	abstract void makeSound();
}
class Dog extends Animal{
	@Override
	public void makeSound(){
		System.out.println("왈왈");
	}
}
class Cat extends Animal{
	@Override
	public void makeSound(){
		System.out.println("야옹");
	}
}

public class main{
	public static void main(String[] args){
		Animal dog = new Dog();
		Animal cat = new Cat();
		
		dog.makeSound();
		cat.makeSound();
	}
}
```
