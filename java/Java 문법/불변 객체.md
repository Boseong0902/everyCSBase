# 불변 객체

불변 객체는 생성 이후 내부 상태가 바뀌지 않는 객체다.

Java의 대표적인 예시는 `String`이다. 문자열 변경처럼 보이는 연산이 실행되어도 기존 객체를 수정하지 않고 새 객체를 만든다.

## 장점

- 상태가 바뀌지 않아 코드 추론이 쉽다.
- 여러 스레드에서 공유해도 동시성 문제가 줄어든다.
- 값 객체, DTO, Record 같은 구조와 잘 맞는다.

## 주의할 점

참조 변수가 `final`이라고 해서 객체 자체가 항상 불변인 것은 아니다.

`final`은 참조 변수가 다른 객체를 가리키지 못하게 막는 것이고, 참조 대상 객체 내부 상태 변경까지 자동으로 막지는 않는다.
