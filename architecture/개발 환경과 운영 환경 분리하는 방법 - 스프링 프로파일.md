# 개발 환경과 운영 환경 분리하는 방법 - 스프링 프로파일

스프링 부트에서 개발 환경과 운영 환경을 분리하는 가장 대표적인 방법

- `application.yml` -> 공통 설정
- `application-dev.yml` -> 개발 서버
- `application-prod.yml` -> 배포 서버

## 예시

### application.yml

```yaml
spring:
  profiles:
    active: dev # 기본적으로 개발 환경을 활성화 (생략 가능)
  application:
    name: my-spring-app
```

### application-dev.yml

```yaml
spring:
  datasource:
    url: jdbc:h2:mem:devdb # 개발용 인메모리 DB
    username: sa
    password:
logging:
  level:
    org.springframework: DEBUG # 개발 시에는 로그를 상세하게
```

### application-prod.yml

```yaml
spring:
  datasource:
    url: jdbc:mysql://prod-db-server:3306/proddb # 운영용 실제 DB
    username: prod_user
    password: secure_password_123!
logging:
  level:
    org.springframework: INFO # 운영 시에는 꼭 필요한 로그만
```

## 그래서 뭘 할 수 있는데?

스프링 부트는 실행 시, 지정된 프로파일 파일(`application-{profile}.yml`)을 읽어와 기본 `application.yml` 위에 덮어쓰기 한다

## 이걸 제어하는 방법

도커 컴포즈 파일에서 env로 주입

```bash
export SPRING_PROFILES_ACTIVE=prod
java -jar my-app.jar
```

## 두 환경의 로직을 분리해야된다면?

코드 레벨에서 프로파일 활용(`@Profile` 어노테이션 + 인터페이스 구현)

예: 결제 기능을 개발 서버와 운영서버에서 다르게 구현하기

```java
public interface PaymentModule {
    void pay();
}

@Component
@Profile("dev")
public class DevPaymentModule implements PaymentModule {
    public void pay() {
        System.out.println("개발용 가짜 모의 결제 진행");
    }
}

@Component
@Profile("prod")
public class ProdPaymentModule implements PaymentModule {
    public void pay() {
        System.out.println("실제 운영용 카드 결제 진행");
    }
}
```

```java
@Service
public class PaymentService {

    // 1. 인터페이스(규격) 타입으로 주입받음
    // 2. 스프링이 프로파일을 확인하고(docker-compose의 env)
    //    DevPaymentModule 또는 ProdPaymentModule 중 하나를 알아서 꽂아줌
    private final PaymentModule paymentModule;

    @Autowired
    public PaymentService(PaymentModule paymentModule) {
        this.paymentModule = paymentModule;
    }

    /**
     * 비즈니스 로직: 주문 및 결제 처리
     */
    public void orderAndPay(Long orderId, int amount) {
        // 1. 주문 검증 및 내부 로직 수행 (공통)
        System.out.println(orderId + "번 주문에 대한 검증을 완료했습니다.");

        // 2. 결제 처리 (환경에 따라 자동으로 알맹이가 바뀜!)
        paymentModule.pay();

        // 3. 결제 완료 후 후처리 (공통)
        System.out.println("주문 및 결제 처리가 최종 완료되었습니다.");
    }
}
```
