# D - 의존 관계 역전

구체적인 개념들에 종속되지 말고, 추상적인 개념들에 종속적으로 코드를 설계한다

즉, 가능한 한 가장 추상적인 클래스에 의존해서 설계를 해야된다.

`animal → bird → predatorBird → Eagle`

이런 식으로 상속중이라면, 가능한 한, `animal`과 가까운 계층의 클래스에 의존하는게 좋다

## 왜?

상위 비지니스 로직이 구현체들에 의존하지 않기 때문에 구현체들의 교체가 용이해지고

테스트가 쉬워지고

여러 구현체들의 일괄적인 관리가 가능해진다
