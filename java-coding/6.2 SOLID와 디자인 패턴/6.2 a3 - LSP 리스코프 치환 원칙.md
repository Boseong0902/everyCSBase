# L - 리스코프 치환

파생 타입, 즉 자식(서브)클래스는 부모(수퍼)클래스를 무조건 대체할 수 있어야 한다

부모 클래스에서의 필드, 메서드의 역할이나 합의가 자식 클래스에서 깨지는 상황이 발생하면 안된다

## 왜?

`List` 등의 컬렉션으로 일괄 처리할 때, 주입되는 객체에 따라 특정 메서드가 기대하지 않은 행동을 하게 될 수도 있음

## 그럼 애초에 구현이 완료된 메서드는 override를 못하게 문법적으로 제한하면 되는거 아닌가?

애초에 합의는 메서드 뿐만 아니라 변수에도 영향을 미칠 수 있음

특정 변수를 바꾸는 메서드를 자식 클래스에서 바꾸는 새 메서드를 정의해버리면,

그 변수를 사용하는 부모에서 정의한 메서드에서 해당 변수를 봤는데 합의에 따르는, 기대하는 값이 아닌, 다른 값이 들어있을 수 있음

또한 메서드의 제한은 `final`로 제한을 걸 수 있음
