# DI와 IoC

DI는 스프링 컨테이너가 지원하는 핵심 개념이다.

각 클래스가 필요로 하는 의존 객체를 직접 생성하거나 검색하지 않고, 스프링 컨테이너가 Bean 설정 정보를 바탕으로 외부에서 주입한다.

## IoC

IoC는 제어의 역전이다.

객체가 자기 안에서 `new` 키워드로 직접 의존 객체를 생성해 사용하는 방식이 아니라, 스프링 컨테이너가 Bean을 생성하고 필요한 의존 객체를 주입한다.

## DI의 효과

DI를 사용하면 강한 결합이 느슨한 결합으로 바뀐다.

코드 단에서 직접 객체 생성과 외부 의존성을 관리하지 않아도 되므로 객체 교체나 확장이 쉬워진다.

## Bean과의 관계

DI는 스프링 컨테이너가 관리하는 [[Spring Bean과 컨테이너|Bean]]들을 서로 연결하는 방식이다.

Bean 등록과 생명주기 관리는 컨테이너가 맡고, 개발자는 필요한 의존성을 생성자나 필드, 메서드 파라미터로 선언한다.
