# Spring 패러다임

Spring을 이해할 때 먼저 잡아야 하는 축은 객체 생성과 의존 관계를 누가 관리하는지, 그리고 공통 관심사를 어디에 둘 것인지다.

## 핵심 패러다임

- [[DI와 IoC]]: 객체 생성과 의존성 연결의 제어권을 코드 내부에서 컨테이너로 넘기는 방식
- [[Spring Bean과 컨테이너]]: 스프링이 관리하는 객체와 그 객체들의 생명주기/의존 관계 관리
- [[스프링 AOP]]: 핵심 비지니스 로직과 공통 관심사를 분리하는 방식

## 함께 봐야 하는 객체지향 원칙

- [[DIP - 의존 관계 역전]]: 고수준 모듈이 구체 구현체가 아니라 추상화에 의존하도록 설계
- [[OCP - 개방-폐쇄]]: 기존 코드를 덜 수정하고 구현체 추가로 확장하는 방향
- [[정적 팩토리 메서드와 팩토리 메서드]]: 객체 생성 책임을 사용하는 쪽에서 분리하는 패턴
- [[전략]]: 여러 구현체를 같은 인터페이스로 묶고 실행 시점에 선택하는 패턴
- [[프록시]]: 실제 객체 호출 전후에 공통 동작을 끼워 넣는 구조

## 학습 흐름

1. [[Spring Bean과 컨테이너]]에서 스프링이 무엇을 관리하는지 본다.
2. [[DI와 IoC]]에서 객체 생성/주입 책임이 왜 뒤집히는지 본다.
3. [[DIP - 의존 관계 역전]]과 연결해 비지니스 로직이 왜 인터페이스에 의존해야 하는지 본다.
4. [[스프링 AOP]]에서 공통 관심사를 프록시로 분리하는 방식을 본다.
