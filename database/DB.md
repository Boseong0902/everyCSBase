# DB

데이터베이스 관련 개념을 모아둔 인덱스.

DB는 단순히 SQL 문법보다 넓게 봐야 한다. 데이터 모델링, 조회 방식, 성능 최적화, 트랜잭션 정합성, DBMS별 구현 차이, 캐시 전략이 서로 연결되어 있다.

## 학습 흐름

1. [[키]]로 테이블을 식별하고 연결하는 기준을 잡는다.
2. [[SQL JOIN]]으로 테이블 관계를 조회하는 방식을 이해한다.
3. [[데이터베이스 이상 현상]]과 [[정규화와 역정규화]]로 데이터 모델링의 목적을 이해한다.
4. [[인덱스]]로 조회 성능을 높이는 방식과 쓰기 비용을 함께 본다.
5. [[트랜잭션]]과 [[트랜잭션 격리 수준]]으로 정합성과 동시성 제어를 이해한다.
6. [[MySQL과 PostgreSQL 차이]]로 같은 개념이 DBMS별로 어떻게 구현되는지 비교한다.
7. [[JOIN 성능 한계와 확장 전략]], [[Redis 캐시]], [[SQL vs NoSQL]]로 확장성과 성능 트레이드오프를 본다.
