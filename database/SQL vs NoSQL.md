# SQL vs NoSQL

## SQL

## 장점

- 명확하게 정의된 스키마
- 데이터 무결성 보장
- 데이터 중복 없음

## 단점

- 유연성 없음
  - 초기에 설계한 스키마를 바꾸기가 많이 어려움
- 복잡한 조인으로 인한 성능 저하 가능
- 대체적으로 수직적 확장만 가능

데이터 구조, 즉 스키마는 변하지 않고, 튜플의 추가, 삭제, 수정이 많을 때 유리

데이터 중복이 많이 없으므로 정합성 및 수정 시 성능 저하 문제는 없음

하지만 수평적 확장이 어렵고, 스키마가 정해져있어 유동성이 떨어짐

## NoSQL

## 장점

- 스키마가 없기에 구조적인 한계가 없음
- 추후에 수정도 가능
- 수평적 확장도 가능
- 데이터가 애플리케이션에 최적으로 저장됨

## 단점

- 데이터 중복
