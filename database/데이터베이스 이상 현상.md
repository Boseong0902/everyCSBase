# 데이터베이스 이상 현상

불필요한 데이터 중복으로 인해 릴레이션에 대한 데이터 삽입, 수정, 삭제 연산을 수행할 때 발생하는 부작용

이를 제거하는 과정이 [[정규화와 역정규화|정규화]]

## 삽입 이상

새 데이터를 삽입하기 위해 불필요한 데이터도 함께 삽입해야되는 문제

예를 들어서 회원 데이터와 이벤트 당첨 여부 데이터를 같은 테이블에서 관리한다고 해보자

그럼 이벤트 참가하지 않은 신규 회원 데이터를 삽입하기 위해선, 이벤트 당첨 여부 데이터를 임시로 너허야 한다

## 갱신 이상

중복 튜플 중 일부만 변경하여 데이터가 불일치하게 되는 모순의 문제

예를 들어서 한 테이블에 하나의 유저가 여러 이벤트에 참석하는 것을 포함하기 위해, 해당 유저의 칼럼이 여러개가 있다고 해보자.

근데 그 유저의 이벤트 참여 여부랑은 관계없는 고객 나이 데이터를 변경하고싶다

이때, 여러 개의 해당 고객 칼럼들 중 하나만 변경되면, 데이터 불일치 문제가 발생한다

## 삭제 이상

튜플을 삭제하면 꼭 필요한 데이터까지 함께 삭제되는 데이터 손실의 문제

삽입 이상의 반대개념

즉, 필요 없는 데이터를 삭제하려고 할 때, 필요한 데이터까지 같이 삭제되는 문제

회원 데이터와 이벤트 당첨 여부 데이터를 같은 테이블에서 관리한다고 해보자

이 때, 특정 회원이 이벤트를 취소해서 이벤트 당첨 여부 데이터만 지우고자 한다

하지만, 해당 회원 정보까지 같이 삭제가 되어버렸다

## 이상 현상 발생 원인

속성들 간에 존재하는 여러 종속관계를 하나의 릴레이션에 표현함

즉, 릴레이션의 분해가 필요

이게 정규화
